# QA Checklist

Run before final output:

- Every rule claim has a source URL.
- Status label is one of: Input Required, Eligible, Needs verification, Not eligible.
- Any blocker (`Not eligible`) is backed by official source evidence.
- Absolute dates are used for rule/timeline references.
- Timestamp is shown in `SGT`.
- Missing inputs are clearly listed.
- No admission guarantee language appears.
- Source links are full clickable URLs (`https://...`), not labels only.
- `Facts` and `Inference:` are separated.
- No final eligibility outcome is produced when required inputs are missing.
